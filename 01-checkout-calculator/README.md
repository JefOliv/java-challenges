# Challenge 01: Checkout Calculator 🛒

## 📝 Scenario & Goal
This project simulates a real-world e-commerce backend checkout system. The goal was to process product data, calculate the financial totals based on quantity, verify free shipping eligibility, and display a formatted summary in the terminal.

## ⚙️ Requirements
- Declare variables using correct primitive types (`int`, `double`, `boolean`).
- Follow strict clean code naming conventions (`camelCase`).
- Perform mathematical operations to calculate the total amount.
- Print a formatted summary report in English using string concatenation.

## 🛠️ Core Concepts Practiced
- **Data Types:** `int` for IDs/quantities, `double` for prices, and `boolean` for flags.
- **Naming Conventions:** `PascalCase` for the class (`CheckoutCalculator`) and `camelCase` for variables.
- **Output Formatting:** Text concatenation via the `+` operator.

## 🚀 Execution & Results
The code was successfully refactored and aligned with international market standards:
- Plurals and singular forms adjusted (`quantity` instead of `productQuantities`).
- Currency alignment (`$59.90` instead of `59.90$`).
- Code logic completely implemented in English.
