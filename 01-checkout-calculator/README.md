## English Version

### 📝 Scenario & Goal
This project simulates a real-world e-commerce backend checkout system. The goal was to process product data, calculate the financial totals based on quantity, verify free shipping eligibility, and display a formatted summary in the terminal.

### ⚙️ Requirements
- Declare variables using correct primitive types (`int`, `double`, `boolean`).
- Follow strict clean code naming conventions (`camelCase`).
- Perform mathematical operations to calculate the total amount.

### 🛠️ Core Concepts Practiced
- **Data Types:** `int` for IDs/quantities, `double` for prices, and `boolean` for flags.
- **Naming Conventions:** `PascalCase` for the class and `camelCase` for variables.

## Versão em Português

### 📝 Cenário & Objetivo
Este projeto simula o sistema de checkout no backend de um e-commerce real. O objetivo foi processar dados de um produto, calcular os totais financeiros com base na quantidade, verificar a elegibilidade para frete grátis e exibir um resumo formatado no terminal.

### ⚙️ Requisitos do Desafio
- Declarar variáveis usando os tipos primitivos corretos (`int`, `double`, `boolean`).
- Seguir estritamente as convenções de nomenclatura de código limpo (`camelCase`).
- Realizar operações matemáticas para calcular o valor total.

### 🛠️ Conceitos Principais Praticados
- **Tipos de Dados:** `int` para IDs/quantidades, `double` para preços e `boolean` para validações (verdadeiro/falso).
- **Convenções de Código:** `PascalCase` para o nome da classe e `camelCase` para as variáveis.
