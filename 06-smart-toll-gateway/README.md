# 🛣️ Challenge 06: Smart Toll Gateway
[Português](#português) | [English](#english)

## Português

### 📝 Descrição do Projeto
Este é um **desafio gerado por Inteligência Artificial**, projetado para consolidar na prática os aprendizados e conceitos estudados na plataforma da @Alura sobre leitura de dados com a classe `Scanner` e laços de repetição utilizando a estrutura `for`.

O sistema simula o fechamento de turno de uma cabine de pedágio inteligente em uma rodovia internacional. O software faz a leitura dinâmica do tipo e valor pago por 4 veículos, acumulando o faturamento total e calculando a média das tarifas de forma automatizada no final do processo.

### 🛠️ Competências Praticadas
* **Leitura Dinâmica de Dados (`Scanner`):** Leitura interativa via teclado de dados do tipo `String` e `double`.
* **Tratamento do Buffer do Teclado:** Uso correto do método `.nextLine()` após leituras numéricas para evitar o travamento do cursor (limpeza de buffer).
* **Estrutura de Repetição Controlada (`for`):** Controle exato de loops para entrada repetida de dados de veículos.
* **Acumuladores e Operadores Reduzidos:** Uso do operador `+=` para somar de forma limpa os valores monetários no fluxo do programa.

## English

### 📝 Project Description
This is an **AI-generated challenge**, designed to practically consolidate the learning and concepts studied on @Alura's platform regarding dynamic input reading with the `Scanner` class and loop control structures using the `for` statement.

The system simulates the end-of-shift revenue report for a smart toll booth on an international highway. The software dynamically reads the vehicle type and the toll fee paid for 4 vehicles, accumulating the total revenue and calculating the average ticket fee automatically at the end of the process.

### 🛠️ Key Skills Applied
* **Dynamic Data Input Reading (`Scanner`):** Interactive keyboard reading for `String` and `double` data types.
* **Keyboard Buffer Handling:** Correct use of the `.nextLine()` method after numeric readings to prevent cursor skipping bugs.
* **Controlled Loops (`for`):** Precise loop constraint management to repeatedly collect vehicle input data.
* **Accumulators and Compound Operators:** Using the `+=` operator to safely sum up financial records during execution.
