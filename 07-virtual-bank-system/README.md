# 🏦 Challenge 07: Virtual Bank System
[Português](#português) | [English](#english)

---

## Português

### 📝 Descrição do Projeto
Este projeto é a **resolução do desafio oficial** proposto pela **Alura** no encerramento do curso de Fundamentos de Java. O objetivo da atividade foi consolidar de forma prática e integrada todos os conceitos de lógica de programação estruturada abordados pelos instrutores ao longo do módulo.

O sistema consiste em um terminal de autoatendimento bancário virtual (ATM) interativo, onde o usuário visualiza seus dados iniciais e pode realizar operações financeiras contínuas através de um menu de opções dinâmico até decidir encerrar a sessão.

### 🛠️ Competências Praticadas
* **Resolução de Desafios Acadêmicos (Alura):** Aplicação prática de requisitos de negócios propostos em ambiente de ensino tecnológico.
* **Loops Indeterminados (`while`):** Uso do laço condicional para manter o menu ativo e rodando continuamente enquanto o usuário não digitar a opção de saída.
* **Validação de Fluxo e Segurança:** Implementação de barreiras de proteção (`if/else` aninhados) para impedir que transferências ou saques sejam executados se o saldo da conta for insuficiente.
* **Formatação Monetária Avançada:** Uso de `System.out.printf` com especificadores locais (`R$ %.2f`) para exibir valores monetários decimais sempre formatados com duas casas decimais.

## English

### 📝 Project Description
This project is the **resolution of the official challenge** proposed by **Alura** at the end of their Java Fundamentals course. The purpose of this activity was to practically integrate and consolidate all the structured programming logic concepts taught throughout the module.

The software simulates an interactive Automated Teller Machine (ATM) terminal, displaying initial customer information and enabling continuous financial transactions through a loop-controlled options menu until the session is manually ended.

### 🛠️ Key Skills Applied
* **Academic Challenge Resolution (Alura):** Practical implementation of business requirements proposed in a leading tech-education environment.
* **Indeterminate Loops (`while`):** Managing program lifecycle through an option menu, ensuring execution continues until the user inputs the exit command.
* **Flow Validation & System Security:** Implementation of business security constraints (nested `if/else` checks) to deny money operations when the account balance is insufficient.
* **Advanced Currency Formatting:** Integrating `System.out.printf` with format specifiers (`R$ %.2f`) to guarantee all decimal values print cleanly with exactly two decimal places.
